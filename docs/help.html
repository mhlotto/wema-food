How to contribute

Add restaurants or guides via pull request. Keep entries factual and include sources.

Add or edit a restaurant

  1. Add a new file in data/restaurants/ (kebab-case name, e.g., my-spot.yaml).
  2. Copy the template below, fill it out, and keep entries alphabetized by name within the folder.
  3. Set town (required); neighborhood is optional but encouraged.
  4. Set last_verified to when you confirmed the info and include at least one source.
  5. Optional: run python scripts/build_data.py to regenerate aggregated YAML for local preview. CI does this automatically for deploys.
- name: Example Restaurant
  town: Amherst
  neighborhood: Downtown Amherst
  address: 123 Main St, Amherst, MA
  phone: 413-555-0100
  website: https://example.com
  cuisine: Example cuisine
  price: $$
  ordering:
    - dine-in
    - takeout
  dietary:
    vegetarian_friendly: true
    vegan_options: false
    gluten_free_friendly: false
  hours:
    mon: "11:00-21:00"
    tue: "11:00-21:00"
    wed: "11:00-21:00"
    thu: "11:00-21:00"
    fri: "11:00-22:00"
    sat: "11:00-22:00"
    sun: "12:00-20:00"
  highlight_items:
    - item: Signature dish
      note: Why it is good
  notes: Parking, cash-only, waits, etc.
  comments:
    - Short freeform note
  last_verified: 2024-10-15
  sources:
    - type: visited
      detail: "Oct 2024"
    - type: menu
      detail: https://example.com/menu

Validation

Run the lint script before opening a PR (optional but encouraged):

pip install -r scripts/requirements.txt
python scripts/build_data.py
python scripts/validate.py data/restaurants.yaml

Guides

Create a new file in guides/ (e.g., late-night.md) using the template in guides/README.md. Keep guides concise and link back to spots in the YAML.

Sources and tone

  • Prefer recent visits, menu URLs, or official social posts.
  • Keep descriptions factual; avoid copying menu text verbatim.
